# 列表

列表用来连续展示多条内容。适合做待办、新闻、排行、日程、交易记录这类可以上下或左右滚动的内容。

## 用法

在编辑器底部点加号,添加 **列表**,再把每一行要显示的内容放进去。

常见做法是只放一个根子组件作为“行模板”,然后设置「数据源（循环）」让它按数组重复显示。如果只是固定几行内容,也可以直接放多个子组件。

## 属性

| 字段 | 可填写 | 说明 |
|------|------|------|
| 方向 | `vertical` / `horizontal` | `vertical` 上下滚动;`horizontal` 左右滚动 |
| 间距 | `0` / `8` | 每一项之间的距离 |
| 数据源（循环） | `{items[item]}` | 按数组重复渲染行模板 |
| 初始位置 | `0` / `3` | 打开时从第几项开始显示,从 `0` 开始 |
| 列数 / 行数 | `1` / `2` | 竖向时表示列数;横向时表示行数 |
| 交叉轴对齐 | `start` / `center` / `end` | 当列表项没有占满宽度或高度时,控制它靠前、居中或靠后 |
| 滚动条 | `off` / `auto` / `on` | 控制滚动条显示方式 |
| 边缘效果 | `spring` / `none` | 滚到边缘时是否有回弹效果 |
| 分割线宽度 | `0` / `1` | 大于 `0` 时显示分割线 |
| 分割线颜色 | `#1A000000` / `${lineColor}` | 分割线颜色 |
| 宽 / 高 | `auto` / `100` / `100%` / `${h}` | 列表尺寸。长内容通常需要设置明确高度 |
| 水平偏移 / 垂直偏移 / Z 轴偏移 | `0` / `8` / `${x}` | 只移动显示位置,不改变原本占位 |
| 内边距 / 外边距 | `8` / `8,8,8,8` | 四周留白 |
| 显示 | `true` / `false` / `${show}` | 控制是否显示 |
| 背景色 | `#FFFFFF` / `${color}` | 列表背景 |
| 圆角 | `0` / `8` / `50%` | 背景圆角 |
| 旋转角度 | `0` / `15` / `${angle}` | 顺时针旋转角度 |
| 不透明度 | `0` 到 `1` | `1` 为完全不透明 |

## 例:循环显示待办

JS:

```js
let items = [
  { title: "买咖啡", time: "08:30", done: false },
  { title: "晨会", time: "09:00", done: true },
  { title: "整理账单", time: "20:00", done: false },
]
```

列表配置:

| 字段 | 填写 |
|---|---|
| 方向 | `vertical` |
| 间距 | `8` |
| 数据源（循环） | `{items[item]}` |
| 高 | `100%` |
| 分割线宽度 | `1` |

行模板:

```text
横排
 ├─ 文本,「内容」填 ${item.time},「字号」填 12
 └─ 文本,「内容」填 ${item.title},「字号」填 14
```

## 例:横向滚动

把「方向」填 `horizontal`,「列数 / 行数」填 `1`,每个子组件就会从左到右排列并横向滚动。

适合放日期、标签、快捷入口等横向内容。每个子组件最好设置固定宽度,例如 `72` 或 `96`,这样滚动节奏更稳定。

## 注意事项

:::warning 循环最多显示 50 条
「数据源（循环）」最多展开 50 条。接口返回很多数据时,建议在 JS 里先 `items.slice(0, 50)`。
:::

:::tip 长内容要给高度
如果列表放在卡片里,通常给「高」填 `100%` 或固定数字。没有明确高度时,内容可能只按自身占位显示,看不到滚动效果。
:::

:::tip 首版限制
列表目前不支持分组吸顶、底部吸附和侧滑操作。需要点击交互时,把按钮或可打开链接的容器放进每一项里。
:::
